# よくあるエラーコードとトラブルシューティング

API ゲートウェイを使用して大規模言語モデルサービスにアクセスする際、リクエストで異常が発生した場合、ゲートウェイは対応する HTTP ステータスコード (Status Code) および OpenAI の標準仕様に準拠した JSON エラーレスポンスボディをクライアントに返します。

### 1. 統一エラーレスポンス形式
リクエストエラーが発生した際、ゲートウェイは一律で標準の JSON 形式のエラーボディを返します：
```json
{
  "error": {
    "message": "错误原因详细描述...",
    "type": "invalid_request_error",
    "code": "context_length_exceeded",
    "param": null
  }
}
```

### 2. ステータスコードとトラブルシューティングガイド

* **400 Bad Request (不正なリクエスト形式)**
  * **発生し得る要因**: リクエストペイロード (Payload) が有効な JSON ではない。必須パラメータ（`model` や `messages` など）の不足。パラメータの型が正しくない。
  * **トラブルシューティング方法**: 送信された HTTP リクエストの Body を確認し、プロバイダーの API 標準パラメータ（`max_tokens` など）と照らし合わせてフィールドのスペルミスをチェックします。

* **401 Unauthorized (未認証/認証失敗)**
  * **発生し得る要因**: リクエストヘッダーに `Authorization` が付与されていない、または Bearer Token が不足している。トークン (Token) が無効、あるいはシステムから削除されている。トークン文字列にスペース、改行、または余分な接尾辞が含まれている。
  * **トラブルシューティング方法**: リクエストヘッダーの形式が `Authorization: Bearer sk-xxxxx` であることを確認し、管理画面でそのトークンが有効化されているかチェックします。

* **403 Forbidden (权限なし/クォータ制限)**
  * **発生し得る要因**: トークンの利用可能クォータ (Quota) またはユーザーの利用可能残高が枯渇している。現在のトークンに、リクエストされたモデルの呼び出し権限がチェックされていない。トークンが管理者またはシステムによって無効化されている。
  * **トラブルシューティング方法**: システムのフロントエンドにログインしてトークンの残高を確認します。トークンリストで「利用可能なモデルリスト」に現在のリクエストモデルが含まれているか確認します。

* **404 Not Found (API またはルートが存在しない)**
  * **発生し得る要因**: リクエスト URL パスのスペルミス。呼び出されたモデルに対して、管理画面で有効な「アップストリームチャネル」が構成されていない。紐づいているチャネルノードがすべて「無効」状態に設定されている。
  * **トラブルシューティング方法**: 呼び出しパス（`/v1/chat/completions` など）を確認します。管理画面でそのモデルがアクティブなチャネルに関連付けられているか確認します。

* **429 Too Many Requests (レート制限に到達/頻度制限)**
  * **発生し得る要因**: トークンのレート制限 (RPM / TPM) に達した。下位チャネルに対応する上流プロバイダーの公式な頻度制限のしきい値に達した。
  * **トラブルシューティング方法**: コード側に指数バックオフによる再試行ロジックを追加します。トークンの頻度制限設定を確認するか、管理者に連絡してレート制限の引き上げを依頼します。

* **500 Internal Error (ゲートウェイ内部エラー)**
  * **発生し得る要因**: ゲートウェイのデータベース接続が切断されたかタイムアウトした。プラットフォーム内部でキャッチされていないコードの Panic 例外が発生した。
  * **トラブルシューティング方法**: システム管理者に連絡し、バックエンドサービスのコンテナログを確認して例外の原因を特定します。

* **502 Bad Gateway (アップストリームチャネルサービス利用不可)**
  * **発生し得る要因**: 上流の公式エンドポイント（`api.openai.com` など）への接続タイムアウトまたはネットワーク切断。上流の公式アカウントの残高不足、またはモデルが公式に廃止された。
  * **トラブルシューティング方法**: 返された JSON レスポンス内の `message` の詳細を確認し、管理画面でチャネルの可用性をテストし、チャネル側のネットワーク問題やアカウントの残高不足を排除します。

* **504 Gateway Timeout (ゲートウェイ応答タイムアウト)**
  * **発生し得る要因**: リクエストされたモデルの生成に非常に長い時間がかかり、HTTP 接続がタイムアウトした。
  * **トラブルシューティング方法**: 動画生成や画质超解像などの極めて時間のかかるタスクについては、非同期 API（`/v1/video/generations` など）を使用して送信し、その後タスクステータス確認 API をポーリングして結果を取得します。

### 3. インテリジェントヘルスチェックと再試行メカニズム
1. **障害再試行**: チャネル A でモデルに 502 エラーやネットワーク接続タイムアウトなどの利用不可状態が発生した際、ゲートウェイに複数チャネルが構成されている場合、自動的に予備チャネル B に切り替えてサイレントに再試行を行います。このプロセス全体は、呼び出し側に対して完全に透過的です。
2. **ログ監査**: 成功か失敗かにかかわらず、リクエストごとの所要時間、差し引かれた料金、IP、リクエスト/レスポンス Payload などが完全に使用ログに記録され、異常原因の迅速な追跡に役立ちます。
