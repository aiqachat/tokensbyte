# クイックスタートとインターフェース認証の説明

統一APIゲートウェイへようこそ！本ゲートウェイは、OpenAIプロトコルおよび各大規模言語モデル（LLM）プロバイダーのネイティブプロトコルをサポートしており、自動的なチャネル配信、プロトコル変換、およびスマートな課金・決済を提供します。

### 1. ベースURL (Base URL)
統一されたAPIの基準リクエストアドレスは以下の通りです：
```bash
https://{{domain}}
```
ローカルでのデバッグやプライベートデプロイを行う場合は、システムから割り当てられた対応するポートアドレスを使用できます。

### 2. 安全な認証方式
ゲートウェイは、すべてのAPIインターフェースにおいて厳格な権限チェックを提供しています。リクエストに「APIキー（トークン）」を付加するには、以下のいずれかの方法を使用できます：

1. **標準の Authorization リクエストヘッダー (推奨)**
   HTTPリクエストヘッダーに、標準の Bearer トークンを追加します：
   ```http
   Authorization: Bearer sk-your_token_string_here
   ```

2. **Google プロトコル互換リクエストヘッダー**
   クライアントプログラムが Google ネイティブプロトコルを採用している場合は、公式の API Key リクエストヘッダーを使用できます：
   ```http
   X-Goog-Api-Key: sk-your_token_string_here
   ```

3. **URLパラメータによる受け渡し**
   GET/POSTの単純なリクエストのみをサポートする環境や制限された環境では、URLのクエリパラメータに直接キーを付加することもできます：
   ```bash
   https://{{domain}}/v1/chat/completions?key=sk-your_token_string_here
   ```

> [!IMPORTANT]
> APIキー（トークン）はお客様のアカウントの消費証明（支払情報）となるため、適切に保管し、公開されたクライアント（フロントエンドのHTML/JSなど）にプレーンテキストでハードコードしないでください。バックエンドサービスを介した転送や、環境変数に設定して保存することをお勧めします。
