
### 5. 一般的なエラー応答 (Common Errors)

当プラットフォームは標準の HTTP ステータスコードと統一された JSON エラー形式に従います。以下は一般的なエラーシナリオとトラブルシューティングのヒントです：

*   **400 Bad Request (パラメータエラー / モデルが存在しません)**: リクエストパラメータまたは指定されたモデルが正しくない場合に返されます。
*   **401 Unauthorized (API キーが無効)**: HTTP ヘッダー内の API キーが正しくない、または期限切れの場合に返されます。
*   **429 Too Many Requests (残高不足 / レートリミット)**: クレジット残高が不足しているか、リクエストの頻度が制限を超えている場合に返されます。

*(※ 詳細な JSON エラー構造およびトラブルシューティングのステップについては、英語版 [English (en)] のドキュメントを参照してください)*
