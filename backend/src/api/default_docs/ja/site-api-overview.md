# サイト API 概要と認証

TokensByte プラットフォームのサイト機能 API へようこそ。統一された AI ルートを介して大模型プロバイダーを呼び出すだけでなく、開発者は管理 API を使用して API トークンの管理、ユーザーの課金/使用ログの照会、およびアカウント設定の取得を行うことができます。

### 1. ベース URL
すべてのサイト API エンドポイントのプレフィックスは次のとおりです：
```bash
https://{{domain}}/api/v1
```
*注：公開ルートおよび認証エンドポイントも `/api/v1` の下にネストされています。*

### 2. 認証メカニズム
公開および登録エンドポイントを除き、すべてのサイト管理 API には JSON Web Token (JWT) 認証が必要です。認証するには、次の手順に従ってください：

#### ステップ 1：ログインして JWT トークンを取得する
* **エンドポイント**: `/api/v1/auth/login`
* **メソッド**: `POST`
* **リクエストパラメータ**:
```json
{
  "username": "あなたのユーザー名またはメールアドレス",
  "password": "あなたのパスワード",
  "login_type": "username"
}
```
* `login_type`: オプション。`"username"`, `"email"`, `"mobile"` のいずれかを指定できます。デフォルトは `"username"` です。

* **レスポンスパラメータ**:
```json
{
  "token": "eyJhbGciOi...",
  "user": {
    "id": "1",
    "username": "user1",
    "role": "user",
    "email": "user@example.com",
    "is_active": 1
  }
}
```

#### ステップ 2：リクエストヘッダーに JWT トークンを含める
それ以降のすべての API 呼び出しでは、返されたトークンを標準の `Authorization` ヘッダーに追加します：
```http
Authorization: Bearer <あなたのJWTトークン>
```

> [!IMPORTANT]
> JWT トークンは一時的なものであり、インタラクティブなフロントエンドまたは統合用に設計されています。JWT トークンを漏洩させないでください。モデルへの API 呼び出しに静的トークンが必要な場合は、代わりに **API トークン** 管理エンドポイントを使用してください。
