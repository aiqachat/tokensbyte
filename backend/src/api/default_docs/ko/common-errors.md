
### 5. 일반적인 에러 응답 (Common Errors)

본 플랫폼은 표준 HTTP 상태 코드와 통일된 JSON 에러 응답 형식을 따릅니다. 다음은 대표적인 에러 시나리오 및 해결 방법입니다:

*   **400 Bad Request (파라미터 오류 / 모델 없음)**: 요청 파라미터가 잘못되었거나 지정된 모델이 존재하지 않을 때 반환됩니다.
*   **401 Unauthorized (API 키 인증 실패)**: HTTP 헤더의 API 키가 누락되었거나 만료되었을 때 반환됩니다.
*   **429 Too Many Requests (잔액 부족 / 요청 한도 초과)**: 계정 잔액이 부족하거나 단위 시간당 요청 한도를 초과했을 때 반환됩니다.

*(※ 상세한 JSON 에러 구조와 해결 가이드는 영문판 [English (en)] 문서를 참고해 주시기 바랍니다)*
