# 자주 발생하는 오류 코드 및 문제 해결

API 게이트웨이를 사용하여 거대 모델 서비스에 액세스할 때, 요청 중 예외가 발생하면 게이트웨이는 해당하는 HTTP 상태 코드(Status Code) 및 OpenAI 표준 규격을 준수하는 JSON 오류 응답 본문을 클라이언트에 반환합니다.

### 1. 통일된 오류 응답 형식
요청 오류가 발생하면 게이트웨이는 항상 표준 JSON 형식의 오류 본문을 반환합니다.
```json
{
  "error": {
    "message": "错误原因详细描述...",
    "type": "invalid_request_error",
    "code": "context_length_exceeded",
    "param": null
  }
}
```

### 2. 상태 코드 및 문제 해결 가이드

* **400 Bad Request (잘못된 요청 형식)**
  * **주요 원인**: 요청 페이로드(Payload)가 올바른 JSON 형식이 아님, 필수 파라미터(예: `model` 또는 `messages`) 누락, 파라미터 타입이 올바르지 않음.
  * **해결 방법**: 전송된 HTTP 요청 본문(Body)을 확인하고, 제공사 API 표준 파라미터(예: `max_tokens` 등)와 대조하여 필드 철자를 확인합니다.

* **401 Unauthorized (인증되지 않음/인증 실패)**
  * **주요 원인**: 요청 헤더에 `Authorization`이 포함되지 않았거나 Bearer 토큰이 누락됨, API 키(Token)가 유효하지 않거나 시스템에서 삭제됨, 토큰 문자열에 공백, 줄바꿈 또는 불필요한 접미사가 포함됨.
  * **해결 방법**: 요청 헤더 형식이 `Authorization: Bearer sk-xxxxx`인지 확인하고, 관리자 백엔드에서 해당 토큰이 활성화되어 있는지 점검합니다.

* **403 Forbidden (권한 없음/한도 제한)**
  * **주요 원인**: 토큰의 사용 가능한 할당량(Quota) 또는 사용자의 잔액이 소진됨, 현재 토큰에 현재 요청한 모델의 호출 권한이 선택되지 않음, 토큰이 관리자 또는 시스템에 의해 비활성화됨.
  * **해결 방법**: 시스템 콘솔에 로그인하여 토큰 잔액을 확인하고, 토큰 목록에서 '사용 가능한 모델 목록'에 현재 요청한 모델이 포함되어 있는지 확인합니다.

* **404 Not Found (인터페이스 또는 라우트가 존재하지 않음)**
  * **주요 원인**: 요청한 URL 경로의 철자 오류, 호출한 모델에 대해 백엔드에 유효한 '업스트림 채널'이 설정되지 않음, 하위에 바인딩된 모든 채널 노드가 '비활성화' 상태로 설정됨.
  * **해결 방법**: 호출 경로(예: `/v1/chat/completions`)를 확인하고, 백엔드에서 해당 모델에 활성화된 채널이 연결되어 있는지 확인합니다.

* **429 Too Many Requests (요청 횟수 초과/속도 제한)**
  * **주요 원인**: 토큰의 요청 빈도 제한(RPM / TPM)을 초과함, 하위 채널에 해당하는 업스트림 제공사의 공식 속도 제한 임계값을 초과함.
  * **해결 방법**: 코드 단에 지수 백오프(Exponential Backoff) 재시도 로직을 추가하고, 토큰 빈도 제한 설정을 확인하거나 관리자에게 문의하여 빈도 제한을 상향합니다.

* **500 Internal Error (게이트웨이 내부 오류)**
  * **주요 원인**: 게이트웨이 데이터베이스 연결이 끊어지거나 시간 초과됨, 플랫폼 내부에서 캡처되지 않은 코드 패닉(Panic) 예외가 발생함.
  * **해결 방법**: 시스템 관리자에게 문의하여 백엔드 서비스 컨테이너 로그를 확인하고 예외 원인을 분석합니다.

* **502 Bad Gateway (업스트림 채널 서비스 사용 불가)**
  * **주요 원인**: 업스트림 공식 엔드포인트(예: `api.openai.com`) 연결 시간 초과 또는 네트워크 중단, 업스트림 공식 계정의 잔액 부족 또는 모델이 공식적으로 지원 중단(Deprecated)됨.
  * **해결 방법**: 반환된 JSON 응답 본문의 `message` 상세 내용을 확인하고, 백엔드 채널의 가용성을 테스트하여 채널 네트워크 또는 계정 잔액 부족 문제를 배제합니다.

* **504 Gateway Timeout (게이트웨이 응답 시간 초과)**
  * **주요 원인**: 요청한 모델의 생성 시간이 극도로 길어 HTTP 연결이 시간 초과됨.
  * **해결 방법**: 비디오 생성이나 화질 개선(Super Resolution) 등 고도로 오랜 시간이 소요되는 작업은 비동기 API(예: `/v1/video/generations`)를 통해 제출하고, 이후 작업 상태 조회 API를 폴링하여 결과를 가져옵니다.

### 3. 스마트 헬스 체크 및 재시도 메커니즘
1. **장애 재시도**: 채널 A에서 모델 호출 시 502/네트워크 연결 시간 초과 등의 사용 불가 상황이 발생하면, 게이트웨이에 다중 채널이 설정되어 있을 경우 자동으로 예비 채널 B로 전환되어 백그라운드에서 재시도를 진행하며, 이 모든 과정은 호출단에 완전히 투명하게(Transparent) 처리됩니다.
2. **로그 감사**: 요청 성공 또는 실패 여부에 관계없이, 매 요청의 소요 시간, 차감 금액, IP, 요청/응답 페이로드(Payload) 등이 사용 로그에 완전히 기록되어 예외 원인을 빠르게 추적할 수 있도록 돕습니다.
